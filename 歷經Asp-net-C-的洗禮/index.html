歷經Asp.net C#的洗禮

C# 的新語法體驗

歷經上次發hexo的文章已經過了一年

這一年實在是經歷了太多

在新語法C#和ASP.net的世界纏鬥了一年

雖然說我目前還是一個連專案建起來都不知道為什麼會跟自己的資料庫連不上的菜雞

但工作多數是改現成的東西，還是小有心得

來記錄一下PHP轉到C#碰到的難處吧

1. 型態

第一次碰到這個狀況同樣是在EXCEL產出回應的時候碰到的(是說我跟EXCEL還真有緣分，兩種語法都有寫到)

在PHP中，幾乎$變數就可以解決所有的問題

這是可以在PHP中被使用的

但

C#就必須將寫法改成這樣

要有兩種型態的變數存在

而且int不能變成string，所以在C#中會有許多ToString、ToList、Int32.parse.()

等等的資料轉換

在變數上處理比較嚴謹，也比較麻煩

2. 開發工具

Laravel 與 ASP.net

在開發工具上就有 Visual Studio Code 及 Visual Studio 2019 上的不同方式

雖然沒有規定用哪個程式來撰寫，但支援及方便度還是會有些許差異

就已選相同字元的快捷鍵來說，習慣了在Visual Studio Code 用 Ctrl+D 的我

當我到2019上面發現只會不斷複製同樣的行數出來

而同樣的效果變成了 Alt+Shift+. 才能選上相同字元

會造成自己在開發的速度上減緩許多

必須要適應開發環境使用相對應的快捷鍵

但其實同時也會發現許多酷功能

3. 框架

如同上面所講到的 Laravel 跟 ASP.net

框架上面除了基本的MVC外，有些像是route或是packages.config等等都是框架獨有的功能

當然因為專案的關係基本上都是拿別人現成的 Ctrl+C Ctrl+V 就好

但當碰到設定上的問題的時候

這部分的確會是非常的困擾人

4. SQL

mariaDB與MSSQL的功能雖然大同小異，但資料庫不同 T-SQL 語法就也有明顯的差異

也許之前因為功力不足，MySQL 、 mariaDB都沒有碰到這一塊

但在MSSQL的部分，因專案需求進行了很多Store Procedure的撰寫

其中if後面接的大括弧 { } ，變成了用 BEGIN END 來撰寫

變數宣告也變成了DECLARE @x varchar(10)等等

但資料庫撰寫這些的確讓後端在編譯的時候不用寫入很多SQL語法，增加程式碼上面的安全性

(只是弱掃還是沒過(◍•ᴗ•◍))

5. 測試

再有虛擬機或測試環境的狀況下

PHP只要把修改好的程式碼丟到指定位置

C#也是發佈(Bluid)完成後丟到指定位置

即可運行

但如果是本機測試一小段Code的時候

PHP可以透過PHP Sandbox來測試

C#則是透過2019創造新的C#專案跑cmd看執行結果

其他諸多的差異可能因為自己功力過淺或是暫時沒想到沒撰寫到

如果有發現可以再向我補充，感謝客官滑時間觀賞小弟的分享

對比一年前的自己，看來是著實的進步了一些了呢(˘•ω•˘)
